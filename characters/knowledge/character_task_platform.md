【任务平台与空投、公售之间的关系】

在 Web3 项目发币前,常常会使用任务平台进行社区激励.这些平台会发布交互任务、邀请任务、社交任务等.

虽然项目不会直接说“做任务就能空投”,但它们常通过任务数据来做快照或分配依据.

因此,判断任务质量与背后是否存在“潜在空投”极为关键.

---

【主流任务平台】

| 平台名称 | 说明 |
|----------|------|
| Galxe    | 最常用于积分型任务系统,支持链上数据与社交账号绑定 |
| Zealy    | 轻量化任务系统,以“XP经验”作为衡量标准 |
| TaskOn   | 偏社交任务型,可整合多平台任务(X、Discord、Dapp) |
| Layer3   | 偏链上交互任务,多用于 DeFi 生态任务引导 |
| Dework   | 偏 DAO 工作任务,但部分项目也用其发布参与任务 |

---

【如何判断任务是否“值得做” / “可能发币”】

✅ 以下行为代表**项目正在收集用户画像,有高度可能空投或开启公售:**

🔹 项目将任务与链上地址绑定  
- 登录任务平台后,要求连接钱包  
- 积分系统直接关联钱包地址  
- 任务数据显示 “您已完成 4/6 交互任务” 等说明链上行为有被记录  
- 说明项目有快照打算,可用于空投资格分配

🔹 任务平台上设置“排行榜” / “积分系统”  
- 如 Galxe、Zealy、TaskOn 上出现“Top 100 地址”、“积分排名”  
- 往往代表未来发币/发空投时,会以排名作为权重参考  
- Retroactive(追溯型)空投常依据此类排行榜作为基础数据

🔹 项目任务内容持续更新  
- 非一次性任务:例如每周增加新任务、每轮有新要求  
- 说明项目对社区参与度持续评估,任务被视为真实行为验证  
- 持续发布任务说明项目在推进阶段(快照、Testnet、合约部署)

🔹 官方社媒配合引流任务  
- 项目在 X / Discord 宣传“新任务已上线”、“冲榜赢机会”等  
- 项目越主动推任务,越可能将其用作 TGE 前资格机制

---

【反之,以下信号代表“空投概率低”或“无币项目”】

🚫 项目任务平台只做表单或问卷  
- 无链上交互、无钱包连接,不具备快照能力

🚫 任务系统中无积分 / 排名  
- 用户参与后无记录、无权重分配  
- 多用于冷启动社群,不具备空投导向性

🚫 平台上已长期无更新(>30天)  
- 原有任务未替换或更新,可能是放弃运营或仅引流  
- 这类项目可能暂无 TGE 计划,仅进行数据收集

🚫 项目明确声明“不发币”但仍发布任务  
- 可能以“DAO门槛”、“NFT角色”进行替代激励  
- 不一定没有价值,但非发币向行为

---

【特殊观察:任务类型 vs 空投相关性】

| 任务类型 | 空投可能性 | 说明 |
|----------|------------|------|
| 链上交互(Swap、Bridge) | 高 | 常用于判定真实用户交互 |
| 社交任务(关注、转推) | 中 | 配合链上行为才有意义 |
| 邀请任务 / Referral | 中 | 单独作用弱,防止 Sybil 滥用 |
| 连接钱包签到 | 高 | 快照前常用方式之一 |
| 问卷 / 表单提交 | 低 | 多用于反馈,不直接影响分发 |
| 积分排行榜 | 高 | 可被用作空投分配依据 |

---

【Agent 判断建议】

🧠 如果 Agent 识别到任务平台行为满足以下多项,可判断任务具有“空投/发币可能”:

- 用户需连接钱包 + 任务记录与地址绑定;  
- 平台存在积分制 / 排行榜系统;  
- 任务持续更新(非一次性);  
- 项目在官方社媒中推广该任务活动;  
- 出现“Snapshot”、“Claim”、“积分可兑换”、“排名奖励”等字眼;

📢 此时,Agent 可发布【高质量任务信号】,提醒用户:该项目值得埋伏.

🚨 若任务长期无更新、无地址绑定、无排名系统,建议 Agent 不发布 / 贴上“纯社交任务”标签,不浪费用户精力.
